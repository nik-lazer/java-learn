Имеющиеся проблемы MonitoringService:
1) Небольшое количество неиспользуемого кода:
	protected long collectFromTime = System.currentTimeMillis();
	private static final int DEFAULT_CAPACITY = 1000;
2) Несколько публичных методов startOperation и endOperation с большим количеством параметров, к тому же перегружающих друг друга, что сильно повышает опасность
   ошибиться с порядком параметров (проблема усугубляется используемыми типами параметров - Serializable, Object, что снижает возможность заметить ошибку на
   этапе компиляции)

Для исправления проблем введен класс Operation, методы установки полей которого реализованы в соответствии с их общностью и обязательностью (source и guid)
вынесены в конструктор, start и end объединены вместе (при необходимости можно и разделить), message и params также объединены. В методы getGuid() и getParentGuid()
вынесено приведение к String и обработка NPE, установка isSuccess и isCompleted увязана с логикой старта и завершения операции. Метод parseMessage() перенесен из
сервиса в Operation. Реализован фабричный метод Operation.createOperation для упрощения конструирования объекта.
